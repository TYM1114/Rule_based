# GA_BBS Solver 核心更新說明

本文件旨在說明近期對求解器進行的兩項核心功能更新，這些更新顯著增強了系統的調度和決策能力。

---

## 1. 支援多工作站 (Multi-Workstation)

求解器現在已原生支援擁有多個獨立工作站的倉儲環境，使其能夠適應更複雜、更大規模的場景。

### 功能說明

系統在為「出庫」(Target) 任務尋找最佳目的地時，會遍歷所有設定的工作站及其對應的停靠口 (Port)。它會綜合評估 AGV 的可用時間、行走距離、以及每個獨立停靠口的忙碌狀態，從而為任務選擇最優的 Workstation。

### 關鍵設定

透過在全域設定中傳入 `workstation_count` 即可啟用此功能：

```python
GLOBAL_CONFIG = {
    # ... 其他參數
    'workstation_count': 3,  # 設置系統中的工作站總數
    'port_count': 2,         # 設置每個工作站擁有的停靠口數量
}
```

---

## 2. 智慧回庫演算法 (Enhanced Return-to-Storage Logic)

針對「回庫」(Return/Reshuffle) 任務，系統採用了全新的兩階段啟發式演算法，以實現更智慧的儲位選擇，從而優化貨物佈局並降低未來的搬運成本。

### 決策流程

當一個貨櫃需要從工作站返回儲存區時，求解器會嚴格遵循以下決策流程：

#### 第一階段：過濾無效儲位

系統首先會排除所有不符合條件的儲位 (Stack/Bay)：
1.  **已滿的儲位**：無法再容納新的貨櫃。
2.  **來源儲位**：貨櫃不允許被放回它此次任務的起始儲位。

#### 第二階段：選擇最佳儲位

在所有合規的候選儲位中，系統會依序套用以下規則來找出唯一的最佳解：

1.  **主要規則 - 最小化未來任務**：
    優先選擇**包含最少「未來任務目標箱」的儲位**。這項規則旨在將貨櫃放置在最“不礙事”的位置，以最大限度地減少未來因其造成的堵塞。

2.  **次要規則 - 最小化路徑成本**：
    如果有多個儲位的「未來任務目標箱」數量相同（平手），系統會選擇從當前工作站出發，**AGV 行走路徑最短的儲位**。

這個新的決策模型使得回庫操作更具前瞻性，有效降低了未來不必要的重複搬運，提升了倉儲系統的整體運作效率。
